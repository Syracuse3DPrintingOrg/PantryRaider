A UI password is required by default. Turn off Require authentication only when an outer layer already gates access, such as a reverse proxy (Pangolin, Authelia, Cloudflare Access) or the Home Assistant add-on, which authenticates through HA itself.

Kiosk PIN Optional

A satellite turns the UI password off by default (the main server owns access control). Set a short numeric PIN for a lightweight lock that is easy to enter on the touchscreen. Leave blank for no lock.

Two-Factor Authentication (TOTP) Optional

When a UI password is set, enable TOTP to require a 6-digit code from an authenticator app (Google Authenticator, Authy, Bitwarden, 1Password, etc.) on each login. Works offline, no email or SMS required.

How secrets are stored
  • Saved keys are never sent back to the browser: fields show a placeholder instead.
  • Leave a field blank to keep the stored value; use the button to erase it.
  • settings.json is written with owner-only permissions (0600).
  • Connection-test errors have key material redacted before display.
