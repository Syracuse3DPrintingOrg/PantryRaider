Save your data off-box and keep the app up to date.

Backup

Pantry Raider's data directory (settings, database, staples list) as a zip. Grocy and Mealie data live in separate containers: use scripts/backup.sh on the host for a full snapshot.

By default secrets are stripped (and rclone.conf is omitted) so the file is safe to store off-box; after restoring you re-enter keys here. Only include them if the destination is trusted.
{% if has.auth_password %}
This backup can carry your API keys and passwords, so confirm your current password to download it.
{% endif %}
Restore from a backup zip
Rewrites this app's data (settings, database, staples) from the zip. The current data is copied aside first, and a redacted backup keeps the API keys you already have. Grocy and Mealie data are not touched here. Reload the page after a restore.
{% if is_pi %}
Full restore (Grocy + Mealie)
Restores the entire stack snapshot made by scripts/backup.sh (this app's data, Grocy inventory, and Mealie). Give a device path, or rclone:remote:path to pull from the configured remote first. This stops the container stack and replaces all data; the current data is moved aside (.pre-restore-<stamp>, nothing deleted) and the stack restarts. Distinct from the app-data Restore above.
{% endif %}
Any rclone remote you've configured via docker exec foodassistant-service rclone config. Config is stored at /app/data/rclone.conf.
USB drive backup
Plug a flash drive into this device and Pantry Raider saves backups to a pantryraider-backups folder on it, on a schedule or on demand. The drive just needs to be formatted and mounted; the newest 14 backups are kept, and nothing else on the drive is touched.{% if is_satellite %} On this device a backup holds its settings and local data, so a replacement unit can be restored from it.{% endif %}
